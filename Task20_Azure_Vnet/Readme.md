Task Name:
Azure Virtual Network (VNet)

Objective:
To understand how networking works in Azure by creating a Virtual Network and subnets.

Description:
Azure Virtual Network is a private network in the Azure cloud. It allows Azure resources to communicate securely using private IP addresses. In this task, a Virtual Network was created along with subnets to understand basic network segmentation.

Steps Performed:
1. Logged in to Azure Portal.
2. Opened Virtual Networks service.
3. Created a new Virtual Network.
4. Selected subscription and resource group.
5. Defined an IP address range using CIDR notation.
6. Created subnets inside the Virtual Network.
7. Verified the Virtual Network and subnet configuration.

Configuration Details:
Virtual Network Name:
task20-vnet

Address Space:
10.0.0.0/16

Subnet 1:
Name: subnet-1  
Address Range: 10.0.0.0/24

Subnet 2:
Name: subnet-2  
Address Range: 10.0.1.0/24

Result:
Virtual Network and subnets were created successfully. The network is ready to connect Azure resources like virtual machines.

Conclusion:
This task helped in understanding Azure networking concepts such as Virtual Network, subnets, and IP addressing.
