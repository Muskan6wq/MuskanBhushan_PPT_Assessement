Objective:
Understand Azure storage services.

Description:
A Storage Account was created and Blob, File Share, and Table storage were explored.


Simple Definition:
Azure Storage Account is used to store data in different formats like blobs, files, and tables.

Objective:
Understand how Azure Storage Account works and how to use Table Storage to store data in the form of entities.

Steps Performed:

1. Logged in to Azure Portal.
2. Opened Storage Accounts service.
3. Created a new Storage Account.
4. Opened the created Storage Account.
5. Went to Data storage section.
6. Selected Tables option.
7. Created a new table (example: task19table).
8. Opened the table after creation.
9. Clicked on “Add entity”.
10. Entered Partition Key and Row Key.
11. Added custom properties like item name.
12. Inserted multiple entities into the table.
13. Verified entities using table view.

Entity Details Added:

Entity 1:
- Partition Key: food
- Row Key: 101
- item1: french fries
- item2: Coke

Entity 2:
- Partition Key: food
- Row Key: 2
- item1: cheesy-farmhouse

Result:
Data was successfully stored in Azure Table Storage using entities and properties.

Conclusion:
Azure Table Storage allows structured NoSQL data storage using entities identified by Partition Key and Row Key.
Result:
Azure storage services successfully used.
