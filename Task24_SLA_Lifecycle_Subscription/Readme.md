README – Task 24: SLA, Service Lifecycle and Governance

Task Name:
Azure SLA, Service Lifecycle and Governance

Objective:
To understand Service Level Agreements (SLA), service lifecycle stages, and governance concepts in Azure.

Description:
This task focuses on understanding how Azure guarantees service availability using SLA, how services move through lifecycle stages, and how governance is managed using subscriptions and access control.

Topics Covered:

Service Level Agreement (SLA):
- SLA defines the uptime and availability guarantee of Azure services.
- Higher SLA means higher service reliability.
- Azure provides service credits if SLA is not met.

Service Lifecycle:
- Azure services go through stages like Preview, General Availability (GA), Deprecated, and Retired.
- Preview services are for testing.
- GA services are stable and recommended for production use.

Governance in Azure:
- Governance helps manage and control Azure resources.
- Subscriptions act as billing and access boundaries.
- Access Control (IAM) manages permissions using roles.
- Policies and cost management help enforce rules and control spending.

Conclusion:
This task helped in understanding Azure service guarantees, lifecycle management, and governance practices for secure and controlled cloud usage.
